Front-end 개발시 고려

구분 방법 효과
css at the top 순차적으로 렌더링 되기위함.
avoid expressions expression은 별도의 이벤트 뿐만아니라 마우스의 작은 움직시에도 계속 호출됨.
make external
  • 1. 브라우저에의해 캐쉬되기때문에 더 빠름.
  • 2. Internal은 지속적으로 리퀘스트되기때문에 매시간 다운로드됨.
minify 불필요한 문자제거를 통한 용량 최소화.
*압축방식으로 JSMin, YUI Compressor, Obfuscation가있으나 난독화를 통하여 21%~25%정도 압축되더라도 space, tab, 개행을 줄여 최소화한 것이 더 낫다.
*gZip된 것보다 최소화하여 용량 감소시킨 것이 5%더 감소.
css import @import 보다 link를 통한 참조가 더 빠름.
*특정브라우저에서 랜더링이 정상적으로 이루어지지 않음.
avoid filters(IE6) AlphaImageLoader 필터는 브라우저가 랜더링될 때 멈추게함.
*이미지당이 아닌 요소당 메모리 소비를 증가시킴.
*꼭 필요한 경우 언더바핵(_) 사용.
js at the bottom </body>바로위에 스크립트 선언
1. 병렬적인 다운로드가 퍼포먼스 저해.
2. 상단에 필요한 스크립트가 있으므로 defer를 통한 참조.
*상황에따라 약 80%감소(4초로딩 -> 1초미만로딩)
*예외 경우는 상단에 위치.
makup <script language="javascript" src=""></script> (X)
<script type="text/javascript" src=""></script> (O)
make external 1. 브라우저에의해 캐쉬되기때문에 더 빠름.
2. Internal은 지속적으로 리퀘스트되기때문에 매시간 다운로드됨.
minify / reduce duplicate 1. 불필요한 문자제거를 통한 용량 최소화.
*압축방식으로 JSMin, YUI Compressor, Obfuscation가있으나 난독화를 통하여 21%~25%정도 압축되더라도 space, tab, 개행을 주려 최소화한 것이 더 낫다.
*gZip된 것보다 최소화하여 용량 감소시킨 것이 5%더 감소.
2. 중첩함수제거.
minimize DOM access 1. Cache references to accessed elements.
2. Update nodes 'offline' and the then add them to the tree.
3. Avoid fixing layout with JS.
develop smart Event Handlers 잘 만들어진 스크립트 사용.

Back-end 개발시 고려

구분 방법 효과
gZip gZip으로 문서 압축 도큐먼트 파일은 모두 gZip을 거치도록 설정. 80%감소효과있음.
CDN CDN사용 요청을 분산시켜 과도한 트래픽 줄임.
Expires or a cache control Header header에 기간 설정 1. IIS5.0, IIS6.0에 비슷한 이슈가 있음. -> Filetimestamp:ChangeNumber
2. 서버의 expires header 설정을 통해, 클라이언트의 브라우저에 캐싱되는 도큐먼트나 이미지들이 많아 불필요한 트래픽을 감소시킬 수 있음.
Configure Etags 이미지에 Etag부착 서버상의 컴포넌트와 브라우저의 캐쉬에 있는 컴포넌트가 서로 일치하는지를 알기위한 메커니즘.상황에따라설정.
Flush the buffer early flush()는 부분적으로 준비된 html을 브라우저로 미리 보내어, 서버가 html이외의 영역과 다른 작업을 수행하는 동안에, 미리html 코드를 받아 사용자에게 빠르게 보여줄 수 있음. 상황에따라설정.
